HEXO装修小记（一）

将HEXO部署到GITHUB上

参考网上的教程，将HEXO部署到GIT上，换一个喜欢的主题，开始改动。需要注意的是HEXO的版本问题，在HEXO3.0之后的版本中deploy git是分开的，所以需要自己安装，安装命令如下：npm install hexo-deployer-git --save 如果不自己安装的话，会在部署到GIT过程中报错，提示找不到GIT。
每一次对博客完成修改后都需要重新布置到GITHUB上。不然修改并不会生效。代码为：

  • hexo clean
  • hexo generate
  • hexo deploy

    为HEXO更换NexT主题

    使用
    NexT主题美化自己的HEXO博客。在HEXO根目录下右键GIT Bash Here,使用Git checkout代码克隆最新版本的NexT：git clone https://github.com/iissnan/hexo-theme-next themes/next
    其余的内容推荐阅读NexT的官方文档，进行自主设置。

    为自己的HEXO博客绑定域名

    简单的部署好自己的博客后，可以为自己的博客换上一个独有的域名，简单好记。一个.top的域名在阿里云上4元/年，也花不了多少，而且还有更多的选择。国内的话，阿里云，腾讯云；国外的话可以找狗爹(GoDaddy)。购买好自己的域名后，只需要在域名运营商那里管理自己的域名。为自己的域名添加一个解析，如下：记录类型：CNAME；主机记录：@；记录值：yourgithubname.github.io。如果喜欢www的访问形式也可以再添加一条解析记录，如下：记录类型：CNAME；主机记录：www；记录值：yourgithubname.github.io。
    最后一定要在本地站点目录里的source目录下添加一个CNAME文件，内容是域名地址，不带www和http。填写好了之后再重新部署到GITHUB上。

    为HEXO更换icon

    制作好自己的icon图标后，更名为favicon.ico,然后放置到/source文件夹中，再部署即可。应注意网络延迟的问题。

添加评论功能

NexT主题集成了很多评论系统，任选其一均可。这里选择友言为例。添加步骤如下：

  • 注册友言账号，注册之后会给你一段JS代码，可以不用理会。
  • 注册登录完成后，进入后台管理界面，复制下自己的用户ID。
  • 在主题配置文件中设置UID，找到youyan_uid属性，在: 后添加你的UID即可。

添加阅读次数功能

在文章标题下面添加显示阅读次数的功能，可以让自己的心理得到一定程度上的满足。实际上就是对用户访问该文章界面的统计，实际上就是一串可以修改的数字。阅读次数统计是通LeanCloud实现的。和添加评论功能一样，步骤如下：

  • 注册完成登录，完成必要的认证。
  • 进入控制台，创建新应用，选择开发版即可。
  • 在新建的应用中选择存储，点击“创建Class”创建一个新的类。为了方便将Class类名称写为Counter，并选择无限制选项，单击完成。
  • 选择设置，获得应用Key中的AppID与AppKey内容，进行主题配置文件中leanCloud_visitors属性的设为true，将阅读次数数据与站点关联起来，将AppID和AppKey复制粘贴进来。

    特别提醒

  • 配置文件中，属性后面的值在冒号后一定要有一个半角空格。
  • 每次完成修改后，一定要重新进行部署。
